Task: Fix incorrect save behavior where confirmation text is saved instead of proposed value
Context (read carefully)
This app supports a chat driven workflow where an assistant (“Mark”) proposes structured updates in the chat, and the user confirms via a green “Save” button.
A bug exists where, upon confirmation, the app sometimes saves the user’s confirmation text (e.g. "yes save as business problem") instead of the actual proposed value (e.g. "Increase sales of running shoes; identify how and where to sell more.").
This is not a parsing issue.
It is a save path logic bug.
 
Root Cause (do not skip)
There are two save mechanisms:
1.	Proposal based save (correct path)
o	Mark proposes a field/value
o	User confirms
o	The app should save the proposal payload
2.	Legacy chat save (incorrectly reused)
o	Saves the last user message verbatim
o	This was appropriate before structured proposals existed
o	It is now incorrect for confirmations
The bug happens when the confirmation button triggers logic that derives the value from the last user message instead of the parsed proposal.
 
✅ Core Fix Rule (invariant)
When a parsed proposal exists, the system must NEVER save from chat history.
It must save ONLY the proposal’s field + value.
Confirmation text is control input, not content.

def confirm_proposed_updates():

user, _ = get_session_data(token)

if not user or user["state"] != "active":
return render_error("Session expired. Please log in again.")

study_id = request.form.get("study_id", type=int)
field = (request.form.get("field") or "").strip()
value = (request.form.get("value") or "").strip()

# HARD RULE: proposals must supply value explicitly
if not study_id or not field or not value:
return render_error("Missing proposed update data.")

ok, err = policy_validate_for_save(field, value)
if not ok:
return render_error(err)

conn = get_db()
try:
study = conn.execute(
"SELECT * FROM studies WHERE id = ? AND user_id = ? AND status = 'draft'",
(study_id, user["id"]),
).fetchone()

if not study:
return render_error("Study not found or not editable.")

# ✅ ONLY SAVE FROM PROPOSAL PAYLOAD
policy_apply_save(conn, study_id, field, value)
conn.commit()

finally:
conn.close()

return redirect(url_for("index", token=token, configure=study_id))
