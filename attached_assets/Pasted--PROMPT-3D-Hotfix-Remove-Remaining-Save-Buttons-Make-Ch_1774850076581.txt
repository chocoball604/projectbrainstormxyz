PART 3 — Crash-hardening: /send-chat must not crash the server
==========================================================

3.1 In artifacts/brainstorm/app.py send_chat():
Add a top-level try/except around the entire POST handler body.
If ANY exception occurs:
- Write a safe Mark message into chat that clearly indicates failure
- Redirect back to configure view
- Never allow an unhandled exception to bubble and crash Flask

Use this exact user-facing message on failure (single sentence):
“Mark couldn’t complete that request right now. Please try again.”

3.2 Prevent runaway workers:
Before inserting a new “⏳ Mark is thinking...” placeholder:
- Query chat_messages for this study:
  count of messages where sender='mark' AND message_text == "⏳ Mark is thinking..."
- If count > 0:
  - Do NOT spawn a new worker
  - Instead insert a Mark message:
    “Mark is still working on the previous message. Please wait a moment and refresh.”
  - Return redirect immediately

3.3 Do not spawn worker when integration creds are missing:
In send_chat(), before spawning subprocess, check env vars:
- AI_INTEGRATIONS_OPENROUTER_BASE_URL
- AI_INTEGRATIONS_OPENROUTER_API_KEY
If missing:
- Do NOT spawn worker
- Immediately replace the placeholder message text with fallback coaching nudge (already computed as `fallback`)
This avoids spawning workers that will fail immediately.

3.4 Ensure placeholder is always replaced even on spawn error:
If subprocess.Popen fails (exception):
- Update placeholder message_text to `fallback` immediately
- Commit and close
- Return redirect
