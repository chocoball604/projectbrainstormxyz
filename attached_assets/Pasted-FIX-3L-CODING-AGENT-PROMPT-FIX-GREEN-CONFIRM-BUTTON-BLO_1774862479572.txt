Objective
Fix the issue where valid Business Problem proposals do not show the green “Confirm & Save” button, even though:
•	The proposal parses correctly
•	The proposal is proposal-only and well formatted
•	Mark correctly restates after a failed mixed turn
This must be fixed without weakening confidence gates for other fields.
 
Root Cause (Authoritative)
The system’s confidence scoring logic penalizes uncertainty language globally, including for the business_problem field.
Specifically:
•	The value
“I want to sell more running shoes but I don't know how and where”
contains "i don't know"
•	This matches _UNCERTAINTY_WORDS
•	The score is reduced by -0.40
•	final_confidence < 0.70
•	allow_confirm = False
•	Green button is suppressed
This behavior is correct per current code, but incorrect for Business Problem semantics.
Business Problems are allowed to express uncertainty.

o	decision_to_support

o	definition_useful_insight
No other scoring rules should change.
